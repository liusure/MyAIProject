# Quickstart: 映射流程优化验证

## 验证场景

### 场景 1: 系统需求驱动的映射选择 (US1)

1. 启动前后端服务
2. 打开浏览器访问应用
3. 上传一个包含"课程名称"、"学分"、"开课时间"、"教室"等列的 Excel 文件
4. **验证**：映射预览页面展示：
   - 上方：LLM 已识别的只读映射表格（如"课程名称→name"）
   - 下方：未映射的系统字段行，每行显示：
     - 左侧：系统字段中文名称（如"上课时间"）
     - 中间：用途说明 + 降级影响（如"用于检测课程时间冲突；未映射则课程标记为'时间未指定'"）
     - 右侧：下拉框可选择未识别的文件列（如"开课时间"）
   - 必填字段（课程名称、学分）用红色高亮
5. 在"上课时间"行选择"开课时间"列
6. **验证**：其他系统字段行的下拉框中不再出现"开课时间"
7. 点击"确认导入"
8. **验证**：导入成功，课程数据正确

### 场景 2: LLM 完美映射 (FR-008)

1. 上传一个 LLM 能完全识别的 Excel 文件
2. **验证**：映射预览页面只展示只读映射表格和"所有字段已成功识别"提示
3. **验证**：手动映射区域不展示
4. **验证**："确认导入"和"重新选择文件"按钮仍然可用

### 场景 3: 导入后回退重新映射 (US2)

1. 完成一次导入，看到"成功导入 X 门课程"
2. 点击"返回修改映射"按钮
3. **验证**：页面回到映射预览状态，之前的选择被保留
4. 修改某个映射选择
5. 再次点击"确认导入"
6. **验证**：导入使用新的映射关系，旧数据被替换

### 场景 4: 整体流程回退 (US3)

1. 在映射预览页面点击"重新选择文件"
2. **验证**：回到文件上传初始状态
3. 重新上传文件并完成导入
4. 在导入结果页面点击"重新上传"
5. **验证**：回到文件上传初始状态，旧数据被清除

### 场景 5: 不选择直接导入 (降级)

1. 上传包含未识别列的 Excel
2. 在映射预览页面不修改任何下拉框（保持"不映射"）
3. **验证**：降级说明清晰可见
4. 点击"确认导入"
5. **验证**：导入成功，未映射字段的数据被忽略
