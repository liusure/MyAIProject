# 研究报告：LLM 自动选课系统

## 范围澄清

- **系统不执行真实选课**：系统仅提供推荐和方案管理功能，实际选课操作由学生在学校教务系统中自行完成
- 这意味着无需选课事务管理、并发锁、候补队列等复杂机制
- Enrollment 实体替换为 SavedPlan（收藏方案），记录的是学生保存的推荐方案

## 技术选型决策

### 决策 1：后端框架 — FastAPI

- **选择**：FastAPI
- **理由**：
  - 原生异步支持，满足 100 并发性能要求
  - 自动生成 OpenAPI 文档，符合宪法「文档完整性」要求
  - 内置 Pydantic 数据验证，满足「类型安全优先」要求
  - 支持 SSE 流式响应，适配 LLM 流式输出场景
- **备选方案**：
  - Django：过于重量级，包含大量不需要的 ORM/Admin 功能
  - Flask：缺少原生异步和自动文档生成，需大量扩展

### 决策 2：前端框架 — React + TypeScript

- **选择**：React 18 + TypeScript 5.x + Vite
- **理由**：
  - React 生态丰富，组件化成熟
  - TypeScript 满足「类型安全优先」要求
  - Vite 极速开发体验，支持代码分割满足首屏 < 1.5s 要求
  - 丰富的无障碍访问库（react-aria、radix-ui）
- **备选方案**：
  - Vue：团队 React 经验更丰富
  - Angular：过度工程化，学习曲线陡峭

### 决策 3：数据库 — PostgreSQL 16

- **选择**：PostgreSQL 16 + SQLAlchemy 2.x + Alembic
- **理由**：
  - 成熟的关系型数据库，强数据完整性
  - 支持 JSONB 字段可灵活存储课程时间表（多时段）和对话消息
  - SQLAlchemy 2.x 提供完整的类型注解支持
  - Alembic 管理数据库迁移
- **备选方案**：
  - MySQL：JSON 支持和窗口函数不如 PostgreSQL
  - MongoDB：课程数据强关联，关系型更合适

### 决策 4：LLM 集成 — MiMo API + 策略模式

- **选择**：MiMo API 为主，策略模式支持 OpenAI/Claude 切换
- **理由**：
  - 用户指定 MiMo API 为主要 LLM 提供商
  - 宪法要求 LLM 提供商可插拔，使用策略模式实现
  - LLM 输出为 JSON 结构化格式（澄清确认），便于前端解析
  - 降级方案：LLM 不可用时切换至关键词搜索（FR-08）
- **备选方案**：
  - 单一 LLM 绑定：违反宪法「可扩展性」原则

### 决策 5：认证方案 — Session-based

- **选择**：Session-based 认证（用户选择）
- **理由**：
  - 用户明确选择 Session-based 而非 JWT
  - 区分学生和管理员角色（FR-09）
  - 需要服务端 Session 存储（Redis）
- **影响**：
  - 需引入 Redis 作为 Session 存储
  - 水平扩展需 Session 共享（Redis 天然支持）
- **备选方案**：
  - JWT：无状态，但用户明确排除

### 决策 6：缓存方案 — Redis

- **选择**：Redis
- **理由**：
  - 宪法要求「LLM Token 使用优化：缓存重复查询结果」
  - 相似选课需求可复用 LLM 推荐结果
  - Session 存储需要（认证方案决定）
  - 支持水平扩展
- **备选方案**：
  - 内存缓存（LRU）：单实例可用，但不支持水平扩展

### 决策 7：选课冲突检测算法

- **选择**：基于时间区间重叠检测 + 图算法
- **理由**：
  - 时间冲突：将课程时间转换为时间区间集合，检测重叠
  - 先修冲突：构建有向图，检测不满足的先修关系
  - 通勤冲突：计算校区间距离和时间间隔
  - 宪法要求 < 500ms 检测时间，区间算法 O(n²) 对 1000 门课程足够
- **备选方案**：
  - 暴力穷举所有组合：指数级复杂度，不满足性能要求


### 决策 9：课程数据导入

- **选择**：pandas（CSV）+ openpyxl（Excel），自动识别格式
- **理由**：
  - 用户确认同时支持 CSV + Excel，自动识别格式
  - pandas 强大的数据解析和校验能力
  - openpyxl 提供 Excel 读取支持
- **备选方案**：
  - 仅 CSV：限制了使用场景
  - 手动解析：可靠性不如成熟库

## 所有 NEEDS CLARIFICATION 已解决
