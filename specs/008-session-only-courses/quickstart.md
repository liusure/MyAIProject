# Quickstart: 课程数据仅存会话，推荐历史持久化

## Scenario 1: 上传 → 推荐 → 刷新 → 重新上传

1. 启动后端和前端
2. 访问 `/select`，上传一个包含课程数据的 Excel 文件
3. 在对话框输入"推荐一些计算机相关课程"
4. **预期**: 返回推荐方案（如果有 session_courses），或提示"请先上传课表文件"
5. 刷新浏览器页面
6. **预期**: 课程列表为空（SessionStore 已清空），对话历史仍可查看
7. 在对话框输入"推荐一些数学相关课程"
8. **预期**: 系统提示"请先上传课表文件"，不返回推荐
9. 重新上传课程文件
10. **预期**: 再次获得推荐方案

## Scenario 2: 推荐历史快照完整性

1. 上传课程文件
2. 获得推荐方案
3. 刷新页面（清除会话数据）
4. 查看之前的对话记录
5. **预期**: 推荐方案完整展示，包含方案名、匹配度、学分，以及每个课程的详细信息（名称、学分、教师、时间安排）

## Scenario 3: 未上传直接推荐

1. 直接访问 `/select`（不上传文件）
2. 在对话框输入"推荐课程"
3. **预期**: 系统提示"请先上传课表文件"，不从数据库返回旧课程推荐

## Scenario 4: 服务端无 DB 回退

1. 确认数据库 courses 表中有课程数据（旧版本遗留）
2. 不上传文件，直接请求推荐
3. **预期**: 系统不使用数据库中的课程数据，仍提示"请先上传课表文件"
