# Feature Specification: Cookie 认证 + 课程表视图

**Feature Branch**: `002-cookie-auth-timetable`
**Created**: 2026-04-22
**Status**: Draft
**Input**: User description: "现在导入xls提示：第 0 行：Error: 未登录，注意这个系统不需要登录，只需要通过cookie判断设备即可，修复此问题。此外增加功能，在输出结果的时候可以显示课程表，即周一到周日的课表。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Cookie 设备认证替代登录（优先级：P1）

作为访客，我希望直接使用系统而无需登录，系统通过浏览器 Cookie 自动识别我的设备，让我无门槛地使用选课推荐功能。

**为什么是此优先级**：当前系统因要求登录而阻塞了所有功能（包括课程导入），这是阻断用户体验的根本问题，必须最先修复。

**独立测试**：打开浏览器访问系统，无需任何登录操作即可直接使用对话推荐、课程导入等所有功能。

**Acceptance Scenarios**:

1. **假设** 用户首次访问系统，**当** 用户打开任何页面时，**那么** 系统自动在浏览器中设置唯一标识 Cookie，无需用户输入账号密码
2. **假设** 用户使用课程导入功能，**当** 用户上传 XLS/CSV 文件时，**那么** 系统正常导入数据，不再提示"未登录"
3. **假设** 用户之前访问过系统，**当** 用户再次访问时，**那么** 系统通过 Cookie 识别为同一设备，保留该设备的历史对话和收藏方案
4. **假设** 用户使用对话推荐功能，**当** 用户发送选课需求时，**那么** 系统正常响应推荐结果，不再要求登录

---

### User Story 2 - 周课程表视图（优先级：P2）

作为学生，我希望在查看推荐方案时能看到一张可视化的一周课程表（周一到周日），让我直观地了解课程在一周中的分布情况。

**为什么是此优先级**：课程表视图是选课推荐的增强展示，在认证问题修复后，它直接提升用户对推荐方案的理解和决策效率。

**独立测试**：获取推荐结果后，页面展示课程表，验证每门课程正确显示在对应的星期和时间段上。

**Acceptance Scenarios**:

1. **假设** 用户获取了推荐方案，**当** 推荐结果展示时，**那么** 系统在方案卡片内显示一张周一到周日的课程表，课程按时间和星期定位在对应格子中
2. **假设** 课程表中有时间冲突的课程，**当** 用户查看课程表时，**那么** 冲突的课程在课程表中以醒目的视觉标记（如红色背景）高亮显示
3. **假设** 用户的推荐方案中有多门课程，**当** 用户查看课程表时，**那么** 每门课程在课程表中显示课程名称，且不同课程有不同颜色区分
4. **假设** 某一天没有课程，**当** 用户查看课程表时，**那么** 该天对应的列显示为空白

---

### Edge Cases

- 用户清除 Cookie 后再次访问，系统将其视为新设备，之前的数据关联丢失
- 用户使用隐身模式访问，Cookie 在关闭窗口后失效，数据不保留
- 推荐方案中课程跨多天且时间段有重叠（如同一门课周一和周三都有），课程表需正确显示所有时段
- 课程表在窄屏设备上的显示适配

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须通过浏览器 Cookie 自动识别设备，移除所有登录/登出流程和界面
- **FR-002**: 系统必须在用户首次访问时自动创建设备标识 Cookie（UUID 格式），有效期不少于 1 年
- **FR-003**: 系统的所有 API 端点必须接受 Cookie 设备标识作为身份凭证，不再要求 Session 登录
- **FR-004**: 系统必须移除登录页面和登录表单，首页直接展示功能入口
- **FR-005**: 系统必须在推荐结果页面展示周一到周日的课程表视图
- **FR-006**: 课程表必须以时间（行：8:00-18:00，按小时分段）和星期（列：周一至周日）为轴展示课程
- **FR-007**: 课程表中的每门课程必须显示课程名称，且不同课程使用不同背景颜色区分
- **FR-008**: 课程表中时间冲突的课程必须以视觉警告（如红色边框或红色背景）标出
- **FR-009**: 课程表必须在推荐方案卡片内部展示，每个推荐方案有独立的课程表
- **FR-010**: 课程表在移动端必须可横向滚动，确保窄屏可读

### Key Entities

- **设备标识（Device ID）**: 通过 Cookie 存储的 UUID，用于关联用户的对话记录、收藏方案等数据，替代原有的学生账号体系
- **课程表视图（Timetable View）**: 基于推荐方案中课程的时间数据（星期、起止时间），渲染的二维表格界面

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户首次打开系统到可以使用功能的时间为 0 秒（无需登录步骤）
- **SC-002**: 课程导入功能在无登录状态下成功率达到 100%
- **SC-003**: 用户通过课程表视图理解推荐方案的时间分布不超过 5 秒
- **SC-004**: 课程表中时间冲突的检测准确率和可视化标记准确率均为 100%
- **SC-005**: 100% 的推荐方案展示时均包含课程表视图

## Assumptions

- 系统为公共选课推荐工具，不涉及个人隐私数据，无需严格的身份认证
- 用户使用现代浏览器（Chrome/Firefox/Safari 最近两个大版本），支持 Cookie 和 CSS Grid
- 课程表展示仅针对推荐方案中的课程，不展示学生已选课程
- 设备标识方案依赖浏览器 Cookie，用户清除 Cookie 后数据关联将丢失
- 课程时间数据格式为 `{ day_of_week: 1-7, start_period: "HH:MM", end_period: "HH:MM" }`，与现有数据模型一致
