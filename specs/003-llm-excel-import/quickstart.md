# Quickstart: 学生课表智能导入与推荐

## 场景 1: 标准中文列名 Excel 上传

**前置**: 后端运行在 localhost:8000，前端运行在 localhost:5173

1. 学生打开课程推荐页面
2. 点击"上传课表"按钮，选择中文列名 Excel 文件
3. 系统自动识别列映射，展示预览
4. 学生点击"确认导入"
5. 课程列表出现在页面上
6. 学生在对话框输入"推荐数据结构和算法课程"
7. 系统返回推荐方案，方案卡片内含可视化课表
8. 冲突课程在课表中红色高亮

**验证**:
- `GET /api/v1/import/session/courses` 返回导入的课程
- `POST /api/v1/conversation/chat` 推荐时使用导入课程而非 DB 课程
- 课表中课程位置与导入的时间数据一致

## 场景 2: 混合列名 + 缺失字段

**前置**: 仅有"课程名"和"学分"两列的 Excel

1. 上传文件
2. 系统通过 LLM 识别"课程名"→"name"，"学分"→"credit"
3. 降级报告显示: 缺少 schedule, category, instructor 等
4. 确认导入
5. 输入"推荐课程"
6. 推荐结果按名称匹配，课表中标记"时间未指定"

**验证**:
- 导入成功，无错误阻断
- 推荐可正常工作
- 课表展示正确，无时间课程有特殊标记

## 场景 3: LLM 不可用时回退

**前置**: 未配置 LLM API Key

1. 上传文件
2. 系统回退到固定格式解析
3. 若列名完全匹配 → 成功导入
4. 若列名不匹配 → 提示"请检查列名是否符合要求"

**验证**:
- 不因 LLM 不可用而报错
- 匹配的列名正常导入

## 场景 4: 会话数据不持久化

1. 上传并导入课程
2. 刷新页面
3. 课程数据消失
4. 需重新上传

**验证**:
- `GET /api/v1/import/session/courses` 返回 204
- 前端显示"请先上传课表"

## 场景 5: 冲突检测

1. 上传含时间信息的课程 Excel（两门课时间重叠）
2. 确认导入
3. 输入"同时推荐 A 课和 B 课"
4. 推荐方案中标记时间冲突

**验证**:
- ConflictEngine 返回 time 类型冲突
- 前端课表中冲突时段红色高亮

## 集成测试要点

1. **端到端流程**: 上传 → 分析 → 确认 → 课程展示 → 推荐 → 冲突检测 → 课表可视化
2. **Token 节省**: 验证 LLM 调用仅发送表头+3行，不发送完整文件
3. **降级验证**: 去掉可选列后仍可导入和推荐
4. **会话隔离**: 不同 device_id 的课程数据互不影响
5. **回退机制**: LLM 超时时走固定格式解析
