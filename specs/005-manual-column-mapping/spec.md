# Feature Specification: 未识别列手动映射

**Feature Branch**: `005-manual-column-mapping`
**Created**: 2026-04-23
**Status**: Draft
**Input**: User description: "增加功能，对于自动对应失败的列，给予用户自行选择的权利，使用下拉框对于未对应的列可选择对应后导入，若不选择也可以视为用户不需要此列，增加鲁棒性。"

## User Scenarios & Testing

### User Story 1 - 未识别列手动选择映射 (Priority: P1)

学生上传 Excel 后，LLM 自动完成了部分列的映射，但有些列未被识别（显示在"未识别的列"区域）。学生应该可以通过下拉框为这些未识别的列手动选择对应的系统字段，然后完成导入。

**Why this priority**: 当前未识别的列直接被忽略，用户无法干预。手动映射让用户体验更可控，避免因 LLM 偶尔识别失败而丢失有用数据。

**Independent Test**: 上传一个包含 LLM 未识别列的 Excel，在映射预览页面为未识别列通过下拉框选择系统字段，确认后验证数据正确导入。

**Acceptance Scenarios**:

1. **Given** LLM 识别了"课程名称→name"但未识别"授课教师"列，**When** 用户在"未识别的列"区域为"授课教师"选择下拉框中的"instructor"，**Then** 该列被加入映射关系，确认后教师数据正确导入
2. **Given** 用户未对任何未识别列做选择，**When** 用户点击确认导入，**Then** 未选择的列被忽略，不影响已识别列的正常导入
3. **Given** 用户为多个未识别列分别选择了映射，**When** 用户点击确认导入，**Then** 所有用户选择的映射与 LLM 自动识别的映射合并后一起生效

---

### User Story 2 - 可用系统字段动态过滤 (Priority: P2)

下拉框中只显示尚未被映射的系统字段，已被 LLM 或用户映射过的字段不再出现在下拉选项中，避免重复映射。

**Why this priority**: 防止用户将多个原始列映射到同一系统字段，导致数据覆盖或冲突。

**Independent Test**: LLM 已识别"课程名称→name"，验证未识别列的下拉框中不再包含"name"选项。

**Acceptance Scenarios**:

1. **Given** LLM 已将"课程名称"映射到 name，**When** 用户查看未识别列的下拉框，**Then** name 不在选项列表中
2. **Given** 用户将一个未识别列映射到 instructor，**When** 用户查看其他未识别列的下拉框，**Then** instructor 也不再出现在选项中

---

### Edge Cases

- 用户更改了 LLM 已识别的映射（将已映射的列改为其他字段或取消映射）→ 本功能不涉及修改已有映射，仅处理未识别列
- 下拉框中所有系统字段都已被映射，无可选项 → 下拉框显示"无可用字段"，用户只能忽略
- 用户在选择后又切换为"不映射" → 允许，等同于不选择

## Requirements

### Functional Requirements

- **FR-001**: 系统 MUST 在列映射预览页面的"未识别的列"区域，为每个未识别列提供一个下拉选择框
- **FR-002**: 下拉框选项 MUST 包含所有尚未被映射的系统字段（已有 LLM 映射 + 用户手动映射的目标字段不再出现），以及一个"不映射"的默认选项
- **FR-003**: 用户选择后，系统 MUST 将该映射关系合并到最终映射结果中，与 LLM 自动识别的映射一起参与数据导入
- **FR-004**: 用户未选择（保持"不映射"默认值）的列 MUST 被静默忽略，不产生任何错误或警告
- **FR-005**: 系统 MUST 在用户点击确认导入时，收集所有下拉框的当前选择，构建完整的映射关系发送给后端

### Key Entities

- **ManualMapping**: 用户手动选择的映射关系，包含原始列名(source)和用户选择的系统字段(target)
- **MappingResult**（已有扩展）: 最终映射结果 = LLM自动映射 + 用户手动映射

## Success Criteria

### Measurable Outcomes

- **SC-001**: 用户能够为 100% 的未识别列看到下拉选择框
- **SC-002**: 下拉框中不出现已被映射的系统字段，重复映射率为 0%
- **SC-003**: 用户手动映射的列在导入后数据提取正确率与 LLM 自动映射一致
- **SC-004**: 用户不选择任何映射时，导入流程不受影响，可在 1 秒内完成确认操作

## Assumptions

- 下拉框中显示的系统字段标签使用中文名称，方便学生理解
- 系统字段列表固定为 10 个：课程名称、学分、课程编号、授课教师、容量、上课地点、校区、分类、学期、上课时间
- 手动映射功能仅作用于"未识别的列"区域，不修改 LLM 已识别的映射
- 此功能为前端交互增强，后端确认接口无需修改（复用现有 /import/confirm 接口）
