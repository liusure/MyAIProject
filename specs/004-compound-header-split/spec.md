# Feature Specification: 复合列头拆分

**Feature Branch**: `004-compound-header-split`
**Created**: 2026-04-23
**Status**: Draft
**Input**: User description: "注意此类head:课时/学分,首先将其分为再进行对应"

## User Scenarios & Testing

### User Story 1 - 复合列头自动拆分映射 (Priority: P1)

学生上传的 Excel 课表中存在类似"课时/学分"这样的复合列头（用斜杠分隔两个含义）。系统应能识别这类列头，将其拆分为多个子列头后再分别映射到对应的系统字段，而不是仅将整个列名映射到一个字段。

**Why this priority**: 当前系统将"课时/学分"整体映射到 `credit` 字段，丢失了"课时"信息。如果后续需要使用课时数据，无法获取。拆分后可确保复合列头中的每个含义都被正确识别。

**Independent Test**: 上传包含"课时/学分"列头的 Excel，验证系统将其拆分为"课时"和"学分"两个概念，然后分别对应到合适的系统字段（或至少正确提取学分值）。

**Acceptance Scenarios**:

1. **Given** Excel 文件中有一列"课时/学分"，值为"48/3"，**When** 系统分析列映射，**Then** 系统识别该列包含两个含义，将"学分"部分提取为 `credit=3`，并正确映射列头
2. **Given** Excel 文件中有一列"学分/课时"（顺序相反），**When** 系统分析列映射，**Then** 系统同样正确拆分并提取学分值
3. **Given** Excel 文件中"课时/学分"列的值为纯数字"3"（没有斜杠分隔），**When** 系统处理，**Then** 系统将该值直接作为学分，不报错

---

### User Story 2 - 多种复合列头模式支持 (Priority: P2)

系统不仅处理"课时/学分"这一种复合列头，还应支持其他常见模式，如"学年/学期"、"上课时间/地点"等。

**Why this priority**: 不同学校的 Excel 模板格式各异，支持多种分隔符和模式能覆盖更多用户的实际文件。

**Independent Test**: 上传包含多种分隔符（/、-、|）的复合列头的 Excel，验证系统都能正确拆分。

**Acceptance Scenarios**:

1. **Given** 列头为"学年/学期"，**When** 系统分析，**Then** 将其拆分后分别对应到 `semester` 字段
2. **Given** 列头为"上课时间/地点"，**When** 系统分析，**Then** 将其拆分后分别对应到 `schedule` 和 `location` 字段

---

### Edge Cases

- 复合列头中某一部分不对应任何系统字段时，只映射有对应的部分
- 列头中斜杠是文件路径含义（如"src/main"）而非分隔符时，不应拆分
- 值中也包含斜杠（如学分值为"3/4"表示范围）时，不应误拆值

## Requirements

### Functional Requirements

- **FR-001**: 系统 MUST 在 LLM 列映射之前，对文件列头进行预处理，识别包含分隔符（`/`）的复合列头
- **FR-002**: 系统 MUST 将复合列头拆分为多个子列头，保留原始列头同时生成拆分后的虚拟列头供 LLM 匹配
- **FR-003**: 系统 MUST 在数据提取阶段，对复合列头的值进行智能拆分（如"48/3"拆分为课时48和学分3），根据映射结果提取对应部分
- **FR-004**: 系统 MUST 在无法确定拆分对应关系时（如值为纯数字"3"），将该值归给主映射字段（学分）
- **FR-005**: 拆分后的子列头 MUST 参与正常的 LLM 映射流程，与非复合列头无差别对待

### Key Entities

- **CompoundHeader**: 表示一个被识别为复合的列头，包含原始列名、拆分后的子列头列表、原始值拆分模式
- **ColumnMapping**（已有）: 扩展以支持一个原始列映射到多个系统字段的情况

## Success Criteria

### Measurable Outcomes

- **SC-001**: 包含"课时/学分"列头的 Excel 文件，学分提取正确率达到 100%
- **SC-002**: 复合列头拆分功能不增加 LLM 调用次数（拆分在发送给 LLM 之前完成）
- **SC-003**: 纯数字值（无分隔符）的复合列头字段正确处理率 100%，不产生解析错误
- **SC-004**: 用户上传包含复合列头的 Excel 后，列映射预览中能清晰展示拆分过程

## Assumptions

- 复合列头主要使用斜杠 `/` 作为分隔符
- 复合列头中最多包含两个含义（不会出现 A/B/C 三段）
- 系统字段定义中已有足够的描述信息，LLM 能将拆分后的子列头正确匹配
- 此功能是 ImportAnalyzer 预处理层的增强，不改变 LLM 调用接口
